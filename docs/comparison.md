# 三方实验环境对比分析报告

## 1. 对比对象

本文对比三类操作系统教学实验环境：

1. **rCore-Tutorial / rCore-Tutorial-Test**：传统 rCore 教学实验与配套测试体系，强调按章节逐步实现操作系统核心机制。
2. **原始 TanGram-rCore-Tutorial**：本项目参考环境，采用 `tg-rcore-tutorial-ch1~ch8` 章节 crate 与 `tg-*` 功能组件 crate 的组织方式，强化组件化理解。
3. **我们的改良版实验环境**：在 TanGram-rCore-Tutorial 基础上补齐 ch3/ch4 关键练习功能，并增加验证记录、测试用例设计和对比分析，形成更完整的“实现-测试-反馈”闭环。

## 2. 改良目标

原始 TanGram-rCore-Tutorial 已经具备较好的章节化和组件化结构，但在部分练习功能上仍存在未实现点。例如 ch3 的 `trace` 系统调用、ch4 的 `mmap/munmap` 与基于页表的安全 `trace` 访问尚未完整通过练习测试。

我们的改良目标不是推翻原有结构，而是在保留其组件化优势的基础上，补齐关键实验功能，并通过更明确的测试设计与验证记录降低学习者定位问题的成本。

## 3. 定量对比

| 指标 | rCore-Tutorial / Test | 原始 TanGram-rCore-Tutorial | 我们的改良版 |
|---|---:|---:|---:|

| 用户内存读写 trace | 部分章节涉及 | ch3/ch4 未完整通过 | ch3 直接访问，ch4 页表翻译访问 |
| `mmap` 系统调用 | 地址空间实验重点 | 未完整实现 | 已实现 |

## 4. 定性对比

rCore-Tutorial 更适合作为从零理解操作系统实现过程的主线教程，它强调学生按章节逐步完成内核机制。TanGram-rCore-Tutorial 在此基础上进一步引入 crate 组件化设计，将 SBI、上下文切换、虚拟内存、任务管理、系统调用等能力拆分为可复用组件，有助于学生从“章节代码”过渡到“系统结构”的视角。

我们的改良版实验环境主要增强了两个方面。第一，补齐 ch3/ch4 的关键练习功能，使 `trace`、syscall 计数、`mmap`、`munmap` 等实验目标能够被真实运行和验证。第二，将测试通过率、失败现象、触发原因和补充测试用例整理为文档，使实验环境不仅能运行，还能帮助学习者理解“为什么失败、如何验证、改完后如何证明正确”。

因此，我们的改良版并不是单纯增加代码，而是把教学实验中常见的“实现完成但不知道是否正确”问题，转化为可执行测试和可解释反馈。它更适合作为课程实验、团队协作开发和比赛验收场景中的中间层平台。

## 5. 改良功能说明

### ch3: `trace` 与 syscall 计数

ch3 原本已经支持多道程序、时间片轮转、`yield` 和 `clock_gettime`。改良后新增 `sys_trace`，支持三类操作：

- `trace_request = 0`：读取当前任务用户地址处的一个字节。
- `trace_request = 1`：向当前任务用户地址写入一个字节。
- `trace_request = 2`：查询当前任务指定 syscall 编号的调用次数，并将本次 `trace` 调用计入统计。

该功能帮助学生理解系统调用分发、任务控制块扩展、用户态与内核态参数传递等机制。

### ch4: `mmap/munmap` 与安全 `trace`

ch4 引入地址空间和页表后，用户内存访问不能再像 ch3 一样直接解引用地址。改良版实现了：

- `mmap`：按页映射用户虚拟地址，支持读、写、执行权限。
- `munmap`：解除用户虚拟地址映射，并处理非法范围和未映射区域。
- ch4 版 `trace`：通过页表翻译用户地址，在权限允许时读写用户内存。

这使学生能够更直观地看到页表权限、地址合法性、page fault 和系统调用返回值之间的关系。

## 6. 剩余问题与后续改进

当前改良版已经通过 ch3/ch4 exercise 测试，但仍存在后续可改进空间：

- 编译过程中存在 Rust 2024 `unsafe_op_in_unsafe_fn` 警告，后续可通过显式 `unsafe {}` 块清理。
- 当前测试主要覆盖 ch3/ch4，后续可扩展到 ch5 进程、ch6 文件系统、ch8 同步原语。
- 测试输出仍主要依赖 pattern 匹配，后续可增加结构化测试报告，例如输出测试名、失败原因、关联模块和修复提示。

## 7. 小结

三方对比表明，原始 TanGram-rCore-Tutorial 在组件化教学方面具有优势，但部分练习功能尚未完全闭环。我们的改良版在保留其结构优势的同时，补齐 ch3/ch4 关键系统调用，并通过测试记录和补充测试设计增强了教学实验环境的可验证性与可解释性。
